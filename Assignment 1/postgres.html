Installing & Using PostgreSQL

You will be using PostgreSQL 14.1, an open-source object-relational database management system, for your SQL assignment, project and midterm test. In this guide, you will learn how to install PostgreSQL on your PC and how to use psql, PostgreSQL’s command-line client program to access a running PostgreSQL server.

If you’ve any questions related to the installation or usage of PostgreSQL, please post them on LumiNUS > Forum > Project > Questions on PostgreSQL.

1. Installing PostgreSQL

This section describes how to install PostgreSQL using a third-party interactive installer. If you prefer to install from the source code, refer to https://www.postgresql.org/docs/current/installation.html.

First, download an appropriate installer for PostgreSQL version 14.1 from https://www.enterprisedb.com/downloads/postgres-postgresql-downloads. The installation allows you to configure the following: the installation directory, the components to be installed, the database storage directory, the password for the default database superuser named postgres, the port number for the database server to listen on for connections, and the locale for the database. For the components to be installed, it should be sufficient to install "PostgreSQL Server" and "Command Line Tools". For the remaining configurable parameters, you may simply use the default values. For a step by step guide on the installation process, you may refer to https://www.postgresqltutorial.com/install-postgresql (for Windows/macOS) or https://www.enterprisedb.com/postgres-tutorials/installation-postgresql-mac-os (for macOS). Note that you should install PostgreSQL version 14.1 and do not need to install pgAdmin component.

2. Connecting to PostgreSQL

In this section, we describe how to connect to a running PostgreSQL server using PostgreSQL’s command-line tool named psql. This tool should already be installed along with the PostgreSQL server installation.

For Windows users, you can find the psql application named "SQL Shell (psql)" from the Start menu as shown below:

psql provides an interactive terminal interface to edit/execute SQL commands and view query results. psql can also read commands from a file or from command line arguments. In addition, psql also provides many meta-commands that could be used for writing scripts to automate many tasks.

To connect to a PostgreSQL database using psql, you need to provide the following: the host name and port number of the database server, the database name, and the name of the user you want to connect as. The PostgreSQL installation would have created a default database named postgres and also a PostgreSQL user (role in PostgreSQL parlance) named postgres. Since you’re running both the database server and client program on the same machine (i.e., your PC), the host name and port number would have been configured correctly during the installation process. Thus, you just need to provide the password for the user postgres if you are connecting to the default database as the default user.

The following figure shows the terminal interface after psql is started on a PC running Microsoft Windows; the default psql prompt shows the name of the connected database.

There are two types of commands that you can input using psql: SQL commands (which terminate with a semicolon) that will be sent to the database server for execution, and psql’s meta-commands (which precede with a backslash) that will be executed by psql. For example, to exit psql, use the meta-command \q.

SQL commands may span multiple lines as illustrated by the following CREATE TABLE statement:

postgres=# create table students (
postgres=# sid  integer,
postgres=# name varchar(80)
postgres=# );
CREATE TABLE
postgres=#

In the above, "postgres=#" is the psql’s prompt (which can be configured) when connected to the default database named postgres.

Any SQL command that has been typed but has not yet been sent for execution is stored in a memory buffer called query buffer. The contents of this buffer can be edited by invoking a configurable text editor within psql using the meta-command \e. Note that the default editors for Microsoft Windows and Unix are notepad.exe and vi, respectively; the editor program can be configured using the environment variable PGSQL_EDITOR.

The following table shows some of psql’s basic meta-commands.

Meta-command Meaning

\q

Quit psql.

\h

Display all SQL commands with available syntax help.

\h foo

Display syntax of command named foo. E.g., \h create table

\?

Display all psql meta-commands.

\c foo

Connect to another database named foo.

\cd foo

Changes the current working directory to foo.

\d

List all created tables.

\d foo

List information on relation named foo.

\p

Display the contents of the query buffer; if the current query buffer is empty, display the most recently executed query.

\w foo

Output the contents of the query buffer to the file named foo; if the current query buffer is empty, ouput the most recently executed query to the file named foo.

\r

Clear the query buffer.

\e

Invoke the text editor to edit the contents of the query buffer; if the query buffer is empty, edit the mostly recently executed query.

\e foo

Invoke the text editor to edit the contents of a file named foo. The contents of the edited file will be copied to the query buffer at the end of the edit session.

\o foo

Enable future query results to be saved to the file named foo.

\g

Send the contents of the current query buffer to the server for execution; if the current query buffer is empty, the most recently sent query is re-executed.

\i foo

Reads the contents from the file named foo and sent their contents to the server for execution.

\!

Escapes from the psql session to a sub-shell. The psql session resumes when the sub-shell is exited.

\! foo

Executes the shell command given by foo.

The following example illustrates the execution of some psql meta-commands. Download the SQL script named "pizza.sql" from LumiNUS’s SQL File Folder.

The following meta-command, which reads and executes the commands in the file named "pizza.sql", will create the pizza database used in class.

postgres=# \i pizza.sql

The above command assumes that the file pizza.sql is your current working directory. To find out where is your current working directory, you can execute the psql meta-command \! dir (on Windows) or \! pwd (on MacOS).

If the file pizza.sql is not in your current working directory, there are two options to load and execute pizza.sql. The first option is to explicitly specify the absolute/relative pathname of pizza.sql as illustrated below for Windows (assuming that the file is located at C:/Downloads/pizza.sql):

postgres=# \i 'C:/Downloads/pizza.sql'

The second option is to first change your current working directory to the directory containing the file pizza.sql before executing the meta-command \i pizza.sql. To change your current working directory, use the \cd meta-command as illustrated below (for Windows):

postgres=# \cd 'C:/Downloads'

You can view the contents of the table Likes using SQL’s select statement as follows:

postgres=# select * from likes;

To quit from psql, use the meta-command \q.

3. Environment Variables

This section gives an overview of how psql can be configured using environment variables.

To avoid being prompted for postgres's password each time you start psql, you can store the password in the environment variable named PGPASSWORD. To learn how to set up environment variables in Microsoft Windows 10, you may refer to https://superuser.com/questions/949560/how-do-i-set-system-environment-variables-in-windows-10. Alternatively, you could also store the password in a file; refer to https://www.postgresql.org/docs/current/libpq-pgpass.html for details. To learn about other environment variables, refer to https://www.postgresql.org/docs/current/libpq-envars.html.
